## 一、一句话介绍

**PolicyManus 是一个以企业为主体的主动政策情报 Agent**：企业填一次档案，它就持续地把
分散在各级政府官网的惠企政策、申报资质、申报截止"主动"匹配、归纳、提醒到企业面前，
并能在对话中记住你、替你盯着机会、把一项申报准备成可执行方案。

---

## 二、痛点与定位

中小企业普遍面临三个现实问题：

1. **政策散、找不全**：惠企政策分散在国家、省、市、区各级政府门户，栏目结构各异、更新无通知，企业靠人工刷网站根本盯不过来。
2. **看不懂、对不上**：政策条文晦涩，企业很难判断"我到底符不符合、能不能申报、还差什么"。
3. **错过窗口**：申报通知常常只有 1–2 周窗口，截止日期还埋在正文里，错过就是错过一年。

市面上的"政策工具"大多停留在**搜索 + 关键词订阅**，本质是更花哨的检索框。
PolicyManus 的定位差异在于：**以企业档案为主体、以 Agent 为载体的"主动情报服务"**——
不是你去问它，而是它主动替你把"哪些机会、为什么、还差什么、下一步做什么"想清楚、送上门。

## 三、完整产品闭环

一条以企业为主体的主动服务链路，每一环都已打通并可演示：

```
企业档案①  →  公开政策库②  →  政策匹配③  →  工作台 Feed④
   →  申报截止跟踪⑤  →  资质机会⑥  →  私有政策库  →  简报报告
        ↓（Agent 能力层）
   实体记忆 + 跨会话记忆  ·  主动情报简报  ·  自主申报助手
```

| 环节 | 能力 |
|---|---|
| ① 企业档案 | 每企业一份结构化档案（地区/行业/规模/主营/资质/技术域/经营研发指标），是全链路的"主体" |
| ② 公开政策库 | 逆向各级政府门户接口抓取惠企政策，结构化入库 + 向量化（无锡新吴/上海杨浦/江苏省工信厅…） |
| ③ 政策匹配 | 企业档案 × 政策：结构化命中（jieba 分词）+ 语义召回（pgvector）双路 RRF 融合，给出推荐理由 |
| ④ 工作台 Feed | 物化匹配结果为信息流，未读红点、状态机（未读/已读/已申报/忽略） |
| ⑤ 申报截止 | LLM 从政策正文抽取申报截止日期，临期倒计时提醒（≤3 天红 / ≤14 天琥珀） |
| ⑥ 资质机会 | 结构化资质目录 × 档案做差距分析（达标✓/不达标✗/待确认?），材料/流程指引 |
| 私有政策库 | 企业上传自有政策/通知，租户私有向量空间检索（BYO embedding key） |
| 报告 | 一键导出政策匹配简报 PDF（画像 + 匹配 + 差距 + 临期，内置中文字体） |

## 四、核心亮点：让它成为"真 Agent"的三件套

这是本作品区别于"带搜索框的政策网站"的关键——三项能力让系统从"工具"跃迁为"自主智能体"。

### 1. 两层记忆：它记得你

- **第 1 层·实体记忆**：会话启动时把企业档案渲染成 `<enterprise_profile>` 注入系统提示词。
  Agent 因此始终"知道自己服务于哪家企业"——问"我能申报哪些资质"时直接基于档案分析，**不再反问企业信息**。
- **第 2 层·跨会话记忆**：Agent 具备 `memory_save` 工具，当用户陈述了值得长期记住的事实/偏好
  （"我们今年新增 3 项发明专利"、"回答简洁些"）时主动记入长期记忆；下次**新会话**启动自动召回。
  用户可在「Agent 记忆」页查看/删除。

> 价值：从"每次对话都要重新自我介绍"的 chatbot，升级为"记得你是谁、记得你说过什么"的助手。

### 2. 主动情报：它替你盯着

- Agent 在企业"离线"时自主扫描已匹配的政策/资质/临期机会，**用 LLM 归纳出一份带理由的优先级简报**
  （总览 + 若干情报项：是什么机会、为什么现在值得关注、建议下一步、紧迫度）。
- **每天 04:30 自动为所有已建档企业刷新**（应用内 APScheduler 调度），用户回来即看到"我帮你盯着，本期有这些机会"。
- 无 LLM key 时回退**确定性兜底简报**，保证主动情报始终有产出。

> 价值：从"你问我答"升级为"我主动发现并推送"，真正的"主动情报服务"。

### 3. 自主申报助手：它把事办成

- 当用户下达目标驱动指令"帮我把高新技术企业申报准备好"，Agent 调用 `qualification_apply_plan`
  **一站式产出申报准备方案**：申报条件逐条核验 + 需补齐/待确认缺口 + 主要材料清单 + 时间线与政策依据。
- 对"档案未填的待确认项"引导去补档案，对"需人工确认项"建议结合知识库政策原文核对。

> 价值：从"答疑"升级为"把一句话目标编排成可执行的行动方案"。

### 贯穿三件套的"风险纪律"

政策/资质是高风险信息，作品全程坚持可信度纪律：

- 所有资质门槛、申报截止、匹配结论均带**免责声明 + 末次核对日期**；
- 差距分析"档案缺字段一律判待确认（?），绝不误报不达标"；
- 申报截止"抽不到一律 unknown，绝不编造日期"；
- 数据来源页可逐条溯源到政府官网。

## 五、技术架构

### 5.1 分层架构（DDD 四层 + 依赖倒置）

```
interfaces (HTTP 路由 / Schema / 认证依赖)
    ↓
application (用例编排 / 应用服务)
    ↓
domain (领域模型 / Repository 协议 / Agent 与工具抽象 / 纯函数内核)
    ↑ 实现
infrastructure (SQLAlchemy / Redis / COS / Docker Sandbox / LLM / Embedding / 爬虫 / 调度器)
```

- **领域层零依赖外部框架**：匹配算法、差距分析、记忆渲染、简报归纳等核心逻辑都是**纯函数**，
  可离线单测、不碰数据库；Repository 以 `Protocol` 定义、`UnitOfWork` 统一事务边界，便于替换与测试。
- **288 个离线单元测试**覆盖领域纯函数、工具、应用服务；CI 三项（backend/frontend/integration 真库）全绿。

### 5.2 Agent 引擎

- **Planner + ReAct 双 Agent 流**：Planner 规划计划，ReAct 逐步执行，记忆压缩防上下文腐化。
- **工具集**：文件 / Shell / 浏览器 / 搜索 / 知识库检索 / 资质（list/gap/detail/apply_plan）/ **记忆（save/list）** / MCP / A2A。
- **Agentic RAG**：单聊天窗口 + 工具自主检索，回答带来源（文件名/页码/相似度），经 SSE 渲染来源卡片。
- **可扩展集成**：通过 **MCP**（Model Context Protocol）接专业外部工具，通过 **A2A**（Agent-to-Agent）调用外部 Agent。

### 5.3 RAG / 向量检索

```
文件上传 → COS 原文件 → 解析(PyMuPDF) → 规范化 → 分块 → Embedding → pgvector 写入 → 检索 → Agent 工具 → 带引用回答
```

- 用 **PostgreSQL + pgvector** 承载向量，无需额外引入 Milvus/etcd；向量存储经接口封装保留替换边界。
- **双轨 Embedding**：公开库向量化用平台 key（公开空间），私有库用租户 BYO key（私有空间），SaaS 计费清晰、隔离彻底。

### 5.4 多租户隔离

- 访问令牌携带 `user_id` / `tenant_id` / 角色 → 认证依赖生成 `CurrentUser` → Repository 用 `tenant_id` 过滤。
- 隔离有三层验证兜底：① 手动对撞探针脚本；② **CI 内 endpoint 隔离测试**（跨租户→404、本租户→成功双向断言）；
  ③ **CI 内真库 SQL 层 WHERE 隔离回归**（连真 Postgres+pgvector + alembic + 真仓储）。

### 5.5 主动性基建

- 应用内 **APScheduler** 调度器：每天 04:00 自动重爬申报通知源（保鲜临期提醒）、04:30 自动重算情报简报。
- 复用进程内 DB/Embedding/LLM 连接，best-effort：单次/单租户失败只记 warning，不影响应用。

## 七、创新点总结

1. **产品范式创新**：把政策工具从"搜索框"重构为"以企业为主体的主动情报服务"，形成档案→匹配→Feed→截止→资质→报告的完整闭环。
2. **Agent 能力创新**：两层记忆（实体 + 跨会话）+ 主动情报 + 自主申报，三件套让系统具备"记得你 / 替你盯着 / 把事办成"的真 Agent 特质。
3. **工程严谨性**：DDD 分层 + 纯函数内核 + 288 离线测试 + 三层多租户隔离回归 + CI 三绿。
4. **可信度纪律**：政策高风险场景全程免责声明 + 末次核对 + 待确认不误报 + 来源可溯源。
5. **务实的数据工程**：逆向各级政府门户公开接口抓取（已甄别可逆向源、放弃 WAF 死源），LLM 抽取申报截止并坚持"抽不到不编造"。
