# PolicyManus 演示视频脚本

> 目标时长：**6–8 分钟**。面向比赛评委，**先抛高光、后讲闭环**。
> 录制环境：http://118.196.142.222:8088（或本地 Docker 栈）。**录前请先备好演示账号、企业档案已填、公开库已有政策**（见文末"录前准备"）。
> 全程旁白用普通话；操作放慢、关键结论停留 2 秒便于评委看清。

---

## 结构总览

| 段落 | 时长 | 核心 |
|---|---|---|
| 0 开场 | 0:30 | 一句话定位 + 痛点 |

| 2 政策匹配 + Feed | 1:00 | 被动情报：匹配 + 推荐理由 + 临期 |
| 3 资质问答（实体记忆）★ | 1:00 | Agent 知道你是谁、不反问 |

★ = 高光环节，是评分重点，务必清晰呈现。

## 0. 开场（0:30）

**画面**：落地页 / 工作台首页。

**旁白**：
> "中小企业每年有大量惠企政策和申报资质，但它们散落在国家、省、市、区各级政府官网，
> 更新没有通知、条文看不懂、申报窗口常常只有一两周——靠人工根本盯不过来。
> PolicyManus 把这件事交给一个 AI Agent：企业只填一次档案，它就**主动**替你把
> 哪些机会、为什么、还差什么、下一步做什么，全部想清楚、送上门。"

---

## 1. 企业档案——确立"主体"（0:40）

**操作**：左栏「企业档案」→ 展示已填档案（企业名/地区/行业/规模/主营业务/已有资质/经营研发指标）。
（可快速点一下「✨ 智能提取关键词」展示从主营业务一键挖词。）

**旁白**：
> "一切从企业档案开始。它是整个系统的'主体'——填一次，后面所有功能自动使用，
> 不用反复输入。这也是后面 Agent 能'知道你是谁'的基础。"

---

## 2. 政策匹配 + 工作台 Feed——被动情报（1:00）

**操作**：左栏「工作台」→ 顶部分栏（政策机会/资质机会）→ 指向某条机会的**匹配分 / 命中度 / 语义相似度 / 推荐理由** → 指向一条带**临期倒计时徽章**的项。

**旁白**：
> "系统已经把企业档案和抓取来的政府政策做了匹配——结构化关键词命中加上语义召回双路融合，
> 每条都给出推荐理由：命中了哪些关键词、地区是否匹配、语义相关度多少。
> 申报有截止的，会自动倒计时提醒，临期标红。
> 这些政策数据都来自政府官网、可逐条溯源（**快速点一下左栏「数据来源」**）。"

---

## 3. 资质问答——实体记忆（1:00）★

**操作**：新建会话 → 输入「我的企业能申报哪些资质？」→ 等 Agent 调用 `qualification_list` 返回清单。
再追问「高新技术企业我还差什么？」→ 展示差距分析逐条 ✓/✗/?。

**旁白**：
> "注意这里——我直接问'我能申报哪些资质'，Agent **没有反问我企业叫什么、是什么行业**，
> 而是直接基于档案给出了清单。因为系统把企业档案作为'实体记忆'注入了 Agent 的上下文，
> 它始终知道自己服务于哪家企业。
> 再问'高新技术企业还差什么'，它逐条核验：达标、不达标、以及'档案没填所以待确认'——
> 注意它**绝不会因为你没填就武断说你不达标**，这是我们对政策这种高风险信息的纪律。"

---

## 4. 跨会话记忆——它记得你（1:20）★

**操作**：
1. 在**当前会话**对 Agent 说：「我们今年新增了 3 项发明专利，另外以后回答尽量简洁一些。」→ 展示 Agent 调用 `memory_save`（旁白点出工具卡片）。
2. 左栏「Agent 记忆」→ 展示刚记下的两条记忆条目（记于日期、可删除）。
3. **新建一个会话 B** → 问「结合我们的专利情况，冲哪个资质更合适？」→ Agent 主动用上"3 项发明专利"作答、且回答简洁。

**旁白**：
> "我告诉它一件事实——今年新增 3 项发明专利，还有一个偏好——回答简洁点。
> 它主动把这些记进了'长期记忆'（**指工具卡片**）。我们能在'Agent 记忆'页看到它记住了什么。
> 关键来了——我**新开一个全新的会话**，问它结合专利情况冲哪个资质，
> 它**主动想起**了上一段对话里说的 3 项专利，回答也照我说的保持了简洁。
> 这就是跨会话记忆：它不只是一次性的问答机器人，而是**记得你是谁、记得你说过什么**的助手。"

---

## 5. 主动情报简报——它替你盯着（1:10）★

**操作**：左栏「情报简报」→ 点右上「立即生成」→ 展示生成的简报：总览句 + 若干情报项（标题 / 为什么值得关注 / 建议下一步 / 紧迫度圆点，临期项标红）。

**旁白**：
> "前面都是我主动去问。现在反过来——这是'情报简报'。
> Agent 会自主扫描所有与这家企业匹配的政策、资质和临期申报，
> 用 AI 归纳出一份**带理由的优先级简报**：本期最值得关注的几个机会、为什么、建议怎么做、哪个最紧急。
> 而且它**每天凌晨会自动刷新**——也就是说，企业不用打开系统，Agent 也在替它盯着机会。
> 这才是真正的'主动情报服务'，而不是又一个搜索框。"

---

## 6. 自主申报助手——它把事办成（1:10）★

**操作**：回到聊天 → 输入「帮我把高新技术企业申报准备好」→ 展示 Agent 调用 `qualification_apply_plan` 产出的**申报准备方案**（条件核验 / 需补齐缺口 / 材料清单 / 时间线与依据）。

**旁白**：
> "最后看一个真正'办事'的场景。我给它一个目标——'帮我把高新技术企业申报准备好'。
> 它没有零散地反复问，而是**一站式产出了一份申报准备方案**：
> 哪些条件已达标、还需要补齐和确认哪些、要准备哪些材料、申报窗口和政策依据是什么。
> 对没填的指标，它会提示我去补档案；对需要核对的，它建议结合政策原文。
> 从'答疑'到'把一句话目标编排成可执行的行动方案'——这就是自主申报助手。"

---

## 7. 收尾（0:30）

**画面**：回到工作台 / 或一张三件套示意图。

**旁白**：
> "回顾一下：PolicyManus 用一条完整闭环——企业档案、政策匹配、工作台、申报截止、资质、报告——
> 把惠企政策服务做透；更重要的是三项让它成为'真 Agent'的能力：
> **它记得你、它替你盯着、它把事办成**。
> 全程坚持可信度纪律：所有结论都带免责声明和核对日期、来源可溯源。
> 系统已经用 Docker 全栈部署上线，欢迎现场体验。谢谢！"

## 录前准备清单

- [ ] 演示账号已登录，**企业档案已完整填写**（含经营研发指标，否则差距/方案不好看）。
- [ ] 公开政策库**已有数据**（新组织先点「抓取政策」抓一次；让工作台 Feed 有匹配项）。
- [ ] **配好 LLM key**：否则聊天不可用、情报简报会标"规则归纳"而非"AI 归纳"。
- [ ] 第 4 段记忆演示：先在会话 A 说事实 → 确认「Agent 记忆」页有条目 → **再新建会话 B** 验证召回（顺序不能错，记忆只对新会话生效）。
- [ ] 第 3/6 段：确认会话是在档案填好**之后**新建的（实体记忆只在会话首条消息注入）。
- [ ] 网络/录屏分辨率确认，关键结论处放慢、停留 2 秒。
