# 产品范围

## 产品定位

PolicyManus 面向企业用户提供政策检索、政策解读、企业适配分析和咨询报告生成能力。系统支持多租户，每个企业组织的数据相互隔离。

## 本期核心用户流程

1. 用户注册或登录并进入所属企业。
2. 用户可切换有权限访问的企业租户。
3. 管理员创建知识库并上传政策、企业材料或优秀案例。
4. 系统解析、分块并建立可检索索引。
5. 用户向 Agent 提问，Agent 检索授权知识库并返回带来源引用的回答。
6. 用户基于企业材料和政策证据生成政策咨询报告。
7. 系统通过 Docker Compose 完成可重复部署。

## 十天交付范围

必须完成：

- 登录、注册、Token 刷新、退出和租户切换前端。
- 会话、文件、知识库、文档和报告的租户隔离。
- 知识库创建、文件上传、处理状态、检索和引用。
- PDF、DOCX、TXT、Markdown 的最小文档处理能力。
- 一条可演示的政策咨询报告生成链路。
- 一个政策来源的抓取、去重和入库演示。
- Docker 部署与演示数据。

可以降级：

- 解析失败时允许人工上传 Markdown。
- 爬虫可由管理员手动触发。
- 报告先支持 Markdown 或 DOCX 中的一种格式。
- 优秀案例先作为 `case` 类型知识文档，不做训练。

本期不做：

- 知识图谱和 Neo4j。
- Milvus 集群。
- 多来源通用爬虫平台。
- 复杂部门权限和审批流。
- Agent/Skill 市场。
- 计费、套餐和生产级运营后台。

## 验收口径

- 两个租户不能访问彼此的会话、文件、知识库、政策材料和报告。
- 上传一份政策文件后，可以检索到相关片段。
- Agent 回答展示来源文件及可定位的引用信息。
- 提供企业材料后可以生成结构完整的政策咨询报告。
- 新环境按文档配置后可使用 Docker Compose 启动。
