# PolicyManus 演示步骤文档（完整动线）

更新时间：2026-06-20
适用版本：main + `feat/cross-session-agent-memory`（含跨会话记忆、主动情报 Agent、自主申报助手）

> 这份文档把整个 Agent 串成一条**从注册到主动服务**的线性演示动线。一场完整 demo 约 15–20 分钟；
> 时间紧时可只演 ★ 标记的"高光环节"（企业档案→匹配→资质问答→两层记忆→主动情报→自主申报）。

---

## 1. 注册与企业档案 ★（主体确立）

1. **注册组织**：注册页选「创建组织」→ 填组织名 → 自动成为 owner。
   （可顺带提一句「加入已有组织」是申请加入+管理员审批的另一条线，体现多租户隔离。）
2. **填企业档案**：左栏「企业档案」→ 编辑 → 填企业名、地区（默认无锡新吴区）、行业、规模、主营业务、
   已有资质/技术域/关键词，以及成立日期/人员/营收/研发投入/专利等经营研发指标。
   - 看点：「✨ 智能提取关键词」从主营业务一键挖词补全（提升后续匹配命中度）。
3. **保存**：保存即后台重算工作台 Feed。

> **演示要点**：强调"企业档案是整个主动服务链路的源头，一次填写、后续所有功能自动使用"。

---

## 2. 公开政策库 ②（数据底座）

1. 左栏「公开政策库」→ 浏览已入库政策，按地区/来源筛选，点开看正文/文号/原文链接。
2. 如需现抓：owner/admin 点「抓取政策」→ 抓取中横幅 → 约 90s 后自动刷新（fire-and-forget）。
3. 左栏「数据来源」→ 展示政策来源官网外链 + 收录条数 + 最近更新、资质目录来源（发证机关/政策依据）。

> **演示要点**："数据来自政府官网、可逐条溯源，不是瞎编"——信任感。

---

## 3. 政策匹配 ③ + 工作台 Feed ④ ★（被动情报）

1. 左栏「工作台」→ 顶部分栏（全部/政策机会/资质机会）。
2. 展示每条机会的**匹配分 / 命中度 / 语义相似度 / 推荐理由**（命中关键词、地区匹配、语义相关度）。
3. 临期申报项带倒计时徽章（≤3 天红 / ≤14 天琥珀）。未读红点同步左栏。
4. 点「重新匹配」演示即时重算；点条目看详情。

> **演示要点**：档案 × 公开政策的双路融合（结构化命中 + 语义召回，RRF 排序）。命中度高低取决于档案完善度。

---

## 4. 资质机会 ⑥ + Agent 问答 ★（agentic RAG）

1. 左栏「资质机会」→ 按档案匹配的可申报/接近可申报资质清单，详情含**条件差距分析**（达标✓/不达标✗/待确认?）。
2. **进入聊天**（新建会话），依次问：
   - 「我的企业能申报哪些资质？」→ Agent 直接调 `qualification_list`（**不反问企业信息**，因为档案已注入为实体记忆）。
   - 「高新技术企业我还差什么？」→ `qualification_gap`，逐条达标/不达标/待确认。
   - 「申报高企需要哪些材料？」→ `qualification_detail` + 可结合 `knowledge_base_search` 取政策原文。

> **演示要点**：这是 #59 实体记忆的效果——Agent 知道"我服务于哪家企业"，不再反问。

---

## 5. 私有政策库 + 报告导出（交付物）

1. 左栏「知识库」→ 建私有库、上传企业自有政策/通知文件 → indexed 后可在聊天里被检索（租户私有空间，BYO embedding key）。
2. 工作台「导出简报」→ 生成 PDF（企业画像 + 匹配政策 + 资质差距 + 临期申报，内置中文字体、带免责声明）。

---

## 6. 跨会话记忆 ★（两层记忆，比赛高光之一）

> 体现"真 Agent"：既知道服务于谁（第 1 层），又记得你说过的事（第 2 层）。

1. **第 2 层·写入**：在**会话 A**里对 Agent 说一句值得长期记住的话，例如
   「我们今年新增了 3 项发明专利」或「以后回答请简洁一些，别用列表」。
   - 看点：Agent 主动调用 `memory_save` 把要点记入长期记忆。
2. **查看**：左栏「Agent 记忆」→ 看到刚记下的条目（记于日期、可删除）。
3. **第 2 层·召回**：**新建会话 B**，问与之相关的问题（如「结合我们的专利情况，适合冲哪个资质？」）。
   - 看点：Agent 在新会话里**主动想起**会话 A 里说的专利数 / 答复偏好，无需重复说明。

> **关键边界（演示注意）**：记忆注入只在**会话首条消息**生效，故必须**新建会话 B** 才能召回；
> 在同一会话里改档案/加记忆不回溯。改完档案想让 Agent 用上，也要新建会话。

---

## 7. 主动情报 Agent ★（主动服务，比赛高光之二）

> 从"你问我答"升级到"我主动帮你盯着"。

1. 左栏「情报简报」→ 点右上「立即生成」。
2. 看点：Agent 自主扫描已匹配的政策/资质/临期项，**归纳出带理由的优先级简报**——
   总览一句话 + 若干情报项（标题 + 为什么现在值得关注 + 建议下一步 + 紧迫度圆点），临期项标红 high。
3. 说明：这份简报**每天 04:30 自动为已建档企业刷新**（`BriefingRefreshScheduler`），
   用户离线时 Agent 也在工作；现场演示用"立即生成"即时出一份。
   - AI 归纳（配了 LLM key）vs 规则归纳（兜底）会在卡片上标注，二者都有产出。

---

## 8. 自主申报助手 ★（目标驱动编排，比赛高光之三）

> 从"答疑"升级到"把事办成"。

1. 在聊天里下达**目标驱动**指令：「帮我把高新技术企业申报准备好」。
2. 看点：Agent 调用 `qualification_apply_plan` 一站式产出**申报准备方案**——
   - 【申报条件核验】逐条 ✓/✗/?
   - 【需补齐 / 待确认】不达标项 + 档案未填的待确认项 + 缺前置资质 + 需人工确认项
   - 【主要申报材料】清单
   - 【时间线与依据】申报时间 / 政策依据 / 主要价值
3. 若有"待确认（档案未填）"项，Agent 会提示去「企业档案」补齐；对需核对项建议结合知识库原文。

> **演示要点**：一句话目标 → Agent 自主把差距、缺口、材料、时间线编排成可执行方案，并带风险纪律。

---

## 9. 收尾：可强调的整体卖点

- **以企业为主体的主动情报服务闭环**：档案①→公开库②→匹配③→Feed④→截止⑤→资质⑥→私有库→报告→**两层记忆→主动情报→自主申报**。
- **真 Agent 三件套**：实体记忆 + 跨会话记忆（记得你）、主动情报（替你盯着）、目标驱动编排（把事办成）。
- **可信度纪律**：全程免责声明 + 末次核对 + 数据来源可溯源。
- **多租户隔离**：各组织数据互不可见（有手动对撞探针 + CI 自动化隔离测试兜底）。

## 附：演示中可能踩的坑

| 现象 | 原因 / 处理 |
|---|---|
| 问资质 Agent 反问企业信息 | 该会话在档案填写**之前**就建了。新建会话即可（实体记忆只首条消息注入）。 |
| 新会话想不起上次说的事 | 确认记忆已写入（看「Agent 记忆」页有条目）；召回只对**记忆写入之后新建的会话**生效。 |
| 情报简报为空 | 档案太空或公开库无匹配。先完善档案 + 抓取政策，再「立即生成」。 |
| 简报标"规则归纳"而非"AI 归纳" | 平台/租户未配 LLM key，走了确定性兜底。配 key 后重新生成即为 AI 归纳。 |
| 命中度普遍偏低 | 档案只填了主营业务、标签较空属正常；越完善越准（系统会自动从主营业务挖词补充）。 |
