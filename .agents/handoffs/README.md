# 交接记录

一项工作一个文件，命名格式：

```text
YYYY-MM-DD-issue-编号-简短主题.md
```

复制以下模板：

```markdown
# 任务标题

Issue：
分支：
负责人：
更新时间：

## 目标

一句话描述任务目标和验收结果。

## 已完成

- 只写已经落地的事实。

## 接口与迁移

- API、Schema、迁移和配置变化。

## 验证

- 实际运行过的命令及结果。

## 未完成

- 明确到文件、函数或下一操作。

## 风险

- 当前已知风险和不可破坏的假设。

## 下一步

1. 下一位开发者可以直接执行的第一步。
```

任务合并且共享记忆已更新后，可删除不再有价值的交接文件。
