## 走查发现 + 处理

### #1 注册/工作区困惑 —— 非 bug，是引导/可见性问题（已修）
用户选「加入已有组织」+ 现有企业，进来却落在「{名} 的工作区」、还要重填档案，疑似 bug。
- **真相**（查 `auth_service.py`）：`mode="join"` 语义是"向他人组织发加入申请、待 owner/admin 批准"，
  批准前系统自动建**个人工作区**（`f"{display_name} 的工作区"`，`is_personal=True`）作激活租户，
  申请挂 pending。所以落进空个人工作区、重填档案是**按设计**。
- **对单人给自己公司建工作区的场景是错的**：没有管理员会批准 → 永远卡在个人工作区；正确路径是 `create`。
- **关键澄清**：这恰是**租户隔离正常工作**（join 用户被正确隔离、看不到目标组织数据），**非数据泄漏**。修的是引导。
- **改动**：
  - `register/page.tsx`：模式切换下加场景化说明（create=自己首次建工作区；join=同事已建、申请加入需审批、批准前临时个人工作区）；join 成功 toast 点明临时工作区。
  - 后端 `TenantInfo`（`schemas/auth.py`）+ `_to_tenant_info`（`auth_routes.py`）透出 `is_personal`（领域模型 `Tenant.is_personal` 本就有，仅透出，带默认值向后兼容）；前端 `AuthTenant` 加 `is_personal?`。
  - `app-shell.tsx`：当前激活租户 `is_personal` 时，顶部显**可关闭提示横幅**，解释处境（在等审批/还是想自己建公司→重新注册）。

### #3 抓取"似乎没生效" —— 实为去重，功能正常
API 日志确认 14:03 抓取任务跑了，但全是 `UPDATE policies`（`www.shyp.gov.cn` URL）——杨浦区那 ~60 条
今天早已抓入库，按 `source_url` 去重 → 重抓只更新不新增 → 列表条数不变 → 看着像没生效。**无需改。**
- 附带澄清「为何抓取不能自选地区」：抓取需**逐门户写爬虫**，地区跟数据源走（加地区＝写爬虫），不是自由筛选项。

### #4 工作台布局 + 全局地区（已修）
- **工作台 Feed 拆分**（`feed/page.tsx`）：顶部加机会类型分栏「全部机会/政策机会/资质机会」，
  前端按 `m.type` 过滤已加载列表（状态过滤仍走服务端），政策与资质不再混在一个列表；空态文案分情形。
- **公开政策库默认地区**（`policies/page.tsx`）：首挂载拉 `profileApi.get()`，用 `matchRegionByProfile`
  在可选地区里按**区/县名包含**（回退市名、去「市」后缀）匹配企业所在地，命中即预选该地区；
  `didDefaultRegionRef` 守卫只设一次，用户手动切换后不覆盖。
- **资质页不动**：`/qualifications` 本就按企业档案匹配（含地区门槛、无自由地区下拉），加下拉反而矛盾。
