# 全网赛事发现：Bing 结果质量修复

更新时间：2026-07-16  
分支：`fix/web-discovery-bing-quality`  
状态：本地实现与定向验证完成；未提交、未部署。

## 完成内容

- Bing 请求改为中文市场与语言，并将 `filters` 改为原始参数值，交由 `httpx` 只编码一次，修复原先 `%` 被二次编码导致的日期过滤失效。
- 解析 Bing `/ck/a` 跟踪链接中的 `u` 参数，写入真实原文 URL，确保抓取与 `source_url` 全局去重稳定。
- 全网赛事查询增加赛事/报名意图与常见无效结果排除词；赛事发现跳过 Bing、Google、YouTube、应用商店等低价值域名。
- 新增搜索参数、链接还原、查询构造和候选域名过滤的回归测试。

## 验证

- `pytest` 定向套件：**42 passed**，覆盖搜索适配、赛事发现、政策入库与 Feed。
- `python -m compileall api/app` 通过。
- 真实 Bing smoke test（不写库、不推送）：`创新创业大赛 报名` 返回 10 条中文原文结果，包括中国政府网与全国大学生创业服务网；未再出现之前的日文 YouTube 跟踪链接。

## 后续真机验收

1. 构建/部署本分支后，用 owner/admin 创建一个测试赛事关键词订阅。
2. 手动触发或等待每日重爬，确认真实原文以 `origin=web` 写入赛事中心，并且 `source_url` 为原文地址。
3. 再次触发同一订阅，确认不会新增重复公开赛事或重复 Feed/飞书提醒。
4. 用第二租户订阅相同关键词，确认公开赛事复用但命中记录、Feed 与通知仍保持租户隔离。
