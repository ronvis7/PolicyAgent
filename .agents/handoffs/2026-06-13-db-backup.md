# 远程数据库备份

分支：`chore/db-backup`
更新时间：2026-06-13

## 目标

远程共享 PostgreSQL 作为主力库后，配置定期备份；并能把快照拉一份到本地存档。

## 决策

- **不做实时同步/主从复制**（隧道场景下过重且脆）；用定期 `pg_dump -Fc` 逻辑快照。
- 本地副本为**单向只读存档**，不写回远程、不双向同步。
- 两层：服务器端 cron（主备份）+ 本地拉取脚本（异地存档，只存 dump 文件）。

## 已完成

### 服务器端（主备份）
- 脚本 `/opt/policy-postgres/backup.sh`（仓库版本化副本：`scripts/server/db-backup.sh`）：
  读 `/opt/policy-postgres/.env` 取库名/用户/密码，`docker exec policy-postgres pg_dump -Fc`
  写到 `/opt/policy-postgres/backups/<db>_<ts>.dump`（先 `.partial` 再原子改名），`find -mtime +7 -delete` 轮转。
- cron（root）：`30 3 * * * /opt/policy-postgres/backup.sh >> /opt/policy-postgres/backups/backup.log 2>&1`
- 已试跑成功：`policy_manus_20260613_212632.dump (30K)`。

### 本地（异地存档）
- `scripts/db-dump.ps1`：导入 `.env.remote`，复用已有 SSH 隧道（无则临时开一条用完即关），
  用 `postgres:16` 容器经 `host.docker.internal:15432` 走隧道 `pg_dump -Fc` → `backups/`，保留最近 14 份。
- 已试跑成功：`backups/policy_manus_20260613_211921.dump (29.6 KB)`。
- `.gitignore` 增加 `backups/`、`*.dump`、`*.sql.gz`（快照含真实数据，绝不入库）。

## 恢复

任意快照（自定义格式）：
```
pg_restore -h <host> -p <port> -U policy_app -d <目标库> --clean <file>.dump
```

## 剩余/可选

- 本地 `db-dump.ps1` 想定时可挂 Windows 任务计划程序。
- 进一步异地：服务器 cron 里加 `scp`/对象存储上传一份到另一台机。
- 备份恢复演练（建议在本地空库上 `pg_restore` 验证一次完整恢复）。

## 注意

- 服务器上的 `backup.sh` 与 cron 是直接落在服务器（不在 git）；仓库 `scripts/server/db-backup.sh` 为可追溯副本，改动后需重新部署到服务器。
- 备份脚本依赖容器名 `policy-postgres` 与 `/opt/policy-postgres/.env`，若服务器部署变更需同步更新。
