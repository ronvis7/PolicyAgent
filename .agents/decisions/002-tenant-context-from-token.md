# ADR 002：租户上下文来自访问令牌

状态：已决定

日期：2026-06-09

## 背景

用户可属于多个租户。客户端传入任意 tenant_id 容易造成越权和遗漏过滤。

## 决策

普通业务接口的当前租户来自经过验证的访问令牌。切换租户必须调用专用接口，后端验证 Membership 后重新签发令牌。

## 规则

- 路由通过认证依赖获取 `CurrentUser`。
- 应用服务显式接收 `tenant_id`。
- Repository 查询强制包含租户条件。
- 请求体和查询参数不能覆盖当前租户。
- 平台管理员跨租户操作使用独立管理接口和审计逻辑。

## 后果

前端切换租户后必须替换令牌，并刷新租户相关缓存和页面数据。
